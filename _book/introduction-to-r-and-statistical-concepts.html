1.3.8.3 Break Statement

A break statement is a jump statement that is used to terminate the loop at a particular iteration. The program then continues with the next statement outside the loop(if any). When break is encountered within a loop, the loop immediately ceases execution, and control is transferred to the statement immediately following the loop. In the case of nested loops, break only exits the innermost loop in which it is placed.

# break for loop
for (i in c(3, 6, 23, 19, 0, 21)){
    if (i == 0) { break } 
    print(i)
}
## [1] 3
## [1] 6
## [1] 23
## [1] 19
print("Outside Loop")
## [1] "Outside Loop"

1.3.8.4 Stop() statement

This function is used to halt the execution of the entire R script or function and signals an error condition. When stop() is called, it generates an error message, and the program execution terminates at that point. This is typically used for handling critical errors or invalid conditions that prevent further meaningful execution. In summary: break is a control flow statement for exiting loops. stop() is a function for signaling errors and terminating the entire execution.

stop_example <- function(x) {
      if (x < 0) {
        stop("Input cannot be negative.") # Halts execution and signals an error
      }
      return(log(x))
    }

    stop_example(2) 
## [1] 0.6931472

1.4 Understanding Population and Sample

In statistics, a population is the entire group of individuals or observations we are interested in. A sample is a subset of the population, used to make inferences about the whole.

Example (Biomedical): A hospital wants to know the average blood pressure of all its patients (population), but only measures 100 randomly selected patients (sample).

1.4.1 Simulating a Population and Sample

set.seed(1)  # ensures reproducibility
population <- rnorm(1000, mean = 120, sd = 15)  # simulate systolic BP
test_sample <- sample(population, 50)

mean(population)
## [1] 119.8253
mean(test_sample)
## [1] 121.1004
  • rnorm() generates random values from a normal distribution.
  • sample() draws 50 values randomly.
  • We compare the means to understand sampling variability.

1.5 Random Number Generation

Statistical sampling relies heavily on randomization. Computers generate pseudo-random numbers, which appear random but are determined by a seed.

1.5.1 Uniform Random Numbers

runif(10, min = 0, max = 1)
##  [1] 0.71554579 0.15636656 0.26157572 0.77379137 0.59475907 0.09537672
##  [7] 0.15846793 0.70109071 0.26918273 0.29683006

This generates 10 numbers uniformly distributed between 0 and 1.

1.5.2 Using a Seed

set.seed(2024)
runif(10, min = 0, max = 1)
##  [1] 0.8369425 0.3208675 0.6803633 0.6981731 0.4570092 0.7014203 0.4157110
##  [8] 0.3032091 0.8765881 0.1190482

By setting the seed, the same “random” numbers are regenerated each time, which is crucial for reproducibility in research studies.

1.6 Sampling from a Group

1.6.1 Simple Random Sampling (SRS)

Suppose you want to select members of a research ethics board at random:

officials <- c("G", "L", "S", "A", "T")
sample(officials, size = 2, replace = FALSE)
## [1] "L" "S"

This draws 2 officials without replacement. This is the simplest form of random sampling, often used in pilot studies.
